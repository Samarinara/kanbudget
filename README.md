# Design Doc  
**“Kanban-Budget” – Offline-first, Encrypted, PWA**  
*Target stack: Next.js 15 (App Router), TypeScript, Tailwind, shadcn/ui, Dexie, WebCrypto, Vercel static export.*

---

## 0. 30-Second Pitch
A Trello-like board where each card is a spend/earn item.  
Data lives only in IndexedDB, encrypted with a user passphrase.  
Works offline, installs as PWA, syncs optional encrypted blobs via Vercel Edge.  
No servers, no plaintext in cloud.

## 2. Data Model
```ts
// src/types/index.ts
export interface ITxn {
  id?: number;                       // Dexie auto-increment
  desc: string;                      // “Toilet paper 8-roll”
  amount: number;                    // negative = spend
  date: string;                      // ISO
  img?: string;                      // dataURL WebP ≤100 kB
  note?: string;
  recurring?: {                      // optional
    freq: 'daily' | 'weekly' | 'monthly';
    interval: number;                // every N
    end?: string;                    // ISO or null
  };
  templateId?: number;               // link to reusable template
}

export interface IColumn {
  id: string;                        // "today" | "week" | "month" | "done"
  title: string;
  txnIds: number[];                  // ordered card list
}
```

Dexie schema:
```ts
this.version(1).stores({
  txns: '++id, date, amount',
  columns: 'id',
  templates: '++id, name',
});
```

## 3. Crypto Flow
1. **First run**:  
   - User types passphrase → `deriveKey(pass, salt)` → CryptoKey.  
   - Export key as JWK → wrap with PBKDF2 → download JSON file (“recovery kit”).  
   - Store salt in localStorage (public).  
2. **Subsequent runs**:  
   - Use passphrase + salt → same key.  
3. **On every write**:  
   - Plain object → encrypt → ArrayBuffer → store in `txns.cipher` column.  
4. **On read**:  
   - ArrayBuffer → decrypt → plain object.

**Code stub** (src/lib/crypto.ts)
```ts
export async function deriveKey(password: string, salt: Uint8Array) {
  const key = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(password), 'PBKDF2', false, ['deriveKey']
  );
  return crypto.subtle.deriveKey(
    { name: 'PBKDF2', salt, iterations: 100_000, hash: 'SHA-256' },
    key, { name: 'AES-GCM', length: 256 }, false, ['encrypt', 'decrypt']
  );
}
```

## 4. Kanban Board Logic
Columns are fixed:
- **Today** (date = today)  
- **This Week** (date within 7 days)  
- **This Month** (date within 30 days)  
- **Done** (user drops here to archive)

**Rules**  
- Cards are sorted by `date DESC`.  
- Dragging card to another column **only updates its date** to match column rule (keeps crypto simple; no cross-column duplicates).  
- Recurring engine runs on app start → clones future txns into **Today** if due.

## 5. Recurring Engine
```ts
// src/lib/recurring.ts
export function spawnRecurring(txns: ITxn[]): ITxn[] {
  const spawned: ITxn[] = [];
  const today = startOfToday().toISOString();
  for (const t of txns.filter(t => t.recurring)) {
    const next = nextOccurrence(t.date, t.recurring);
    if (next <= today && !alreadySpawned(t.id!)) {
      spawned.push({ ...t, id: undefined, date: today, recurring: undefined });
    }
  }
  return spawned;
}
```
Call inside `useEffect` when db unlocks; insert cloned txns.

## 9. Settings / Data Portability
Settings page:
- **Change passphrase** → re-encrypt every row (batch in transaction).  
- **Export** → Dexie export → JSON blob → download.  
- **Import** → read file → decrypt each row with old key → insert.  
- **Share encrypted backup** → POST to `/api/share` → get short id → copy url.

## 11. Security Checklist
☐ Never print passphrase to console.  
☐ Zero data in URL/query.  
☐ Use `crypto.randomUUID()` for share ids (unpredictable).  
☐ Content-Security-Policy: `default-src 'self'; img-src 'self' blob: data:`  
☐ `SameSite=Strict` on any cookie (not used yet, but future-proof).
